# Требования к программным модулям

## 1. Общее описание

Система должна состоять из трех основных компонентов:
- `DataProvider` — модуль источника данных;
- `Processor` — модуль обработки и нормализации данных;
- `Orchestrator` — модуль интеграции и управления взаимодействием между компонентами.

## 2. Взаимодействие компонентов

1. `Orchestrator` инициирует получение данных через `DataProvider.get_raw_data()`.
2. Результат передается в `Processor.process(raw_data)`.
3. `Processor` преобразует данные, рассчитывает агрегаты и возвращает структуру результата.
4. `Orchestrator` добавляет служебный признак `integrated` и возвращает итог объединенной информации.

## 3. Функциональные требования

- Источник данных должен возвращать словарь с ключами `items` и `metadata`.
- Процессор должен нормализовать каждый элемент и подсчитать сумму по полю `score`.
- Оркестратор должен проверять тип входных данных и отклонять некорректный формат.
- Результат должен содержать признак интегрированности `integrated: true`.

## 4. Нефункциональные требования

- Модули должны быть независимыми, чтобы обеспечить возможность подмены `DataProvider` и `Processor` в тестах.
- Код должен соответствовать стандартам Python PEP 8 и пройти проверку `flake8`.
- Модульная архитектура должна позволять расширять набор компонентов без изменения базовой логики `Orchestrator`.
