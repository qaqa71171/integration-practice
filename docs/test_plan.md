# План тестирования

## 1. Цели тестирования

- Проверить корректность работы модуля поставки данных.
- Проверить корректность обработки и нормализации данных.
- Проверить интеграцию компонентов и защиту от некорректных входных данных.

## 2. Тестовые наборы и сценарии

### 2.1 Модуль `DataProvider`
- `test_get_raw_data_returns_dict`
  - Ожидается: возвращается словарь.
- `test_get_raw_data_items_has_expected_fields`
  - Ожидается: элемент содержит поля `id`, `name`, `value`.

### 2.2 Модуль `Processor`
- `test_normalize_item`
  - Ожидается: нормализация данных выполняется корректно.
- `test_process_calculates_total_score`
  - Ожидается: сумма `total_score` верна, структура результата соответствует ожиданиям.

### 2.3 Модуль `Orchestrator`
- `test_orchestrator_integration`
  - Ожидается: компоненты интегрируются, итоговый результат содержит `integrated: true`.
- `test_orchestrator_rejects_invalid_raw_data`
  - Ожидается: выбрасывается `TypeError` при некорректном формате данных.

## 3. Средства тестирования

- `pytest` для выполнения автоматизированных тестов.
- `flake8` для проверки соответствия стандартам кодирования.

## 4. Запуск тестов

```bash
pip install -r requirements.txt
pytest
flake8 src tests
```
