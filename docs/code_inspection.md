# Инспектирование компонентов на соответствие стандартам кодирования

## 1. Проверка кода

Статическая инспекция проводится с помощью `flake8`.

Проверяется:
- стиль оформления кода (PEP 8);
- максимальная длина строки;
- отсутствие грубых синтаксических ошибок;
- соответствие именования классов, методов и переменных.

## 2. Выдержки стандартов

- Имена классов используют стиль `CamelCase`.
- Имена функций и переменных используют `snake_case`.
- Модули имеют документацию на уровне заголовка и ключевых классов.
- Входные и выходные структуры описаны в коде и должны быть предсказуемыми.

## 3. Результат инспектирования

В текущем проекте:
- модуль `DataProvider` возвращает детерминированные данные;
- модуль `Processor` корректно нормализует элементы и считает сумму;
- модуль `Orchestrator` выполняет проверку типа входных данных.

> Рекомендуется выполнять `flake8 src tests` перед коммитом изменений.
